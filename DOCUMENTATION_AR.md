# توثيق مشروع Venom Desktop

## مقدمة
مشروع **Venom Desktop** هو مدير سطح مكتب (Desktop Manager) حديث مبني باستخدام إطار عمل **Flutter** وموجه لأنظمة **Linux**. يهدف المشروع إلى توفير تجربة مستخدم عصرية، سلسة، وقابلة للتخصيص، مع دعم ميزات متقدمة مثل خلفيات الفيديو وترتيب الأيقونات الحر.

## المتطلبات (Requirements)
لتشغيل المشروع وتطويره، تحتاج إلى تثبيت الحزم التالية على نظام Linux (مثل Ubuntu/Debian):

```bash
sudo apt install libmpv-dev gdk-pixbuf-thumbnailer ffmpegthumbnailer
```

كما يتطلب وجود **Flutter SDK** مثبتًا (الإصدار 3.9.2 أو أحدث).

## التثبيت والتشغيل (Installation & Running)
1.  قم باستنساخ المستودع (Clone Repository).
2.  احصل على الحزم اللازمة:
    ```bash
    flutter pub get
    ```
3.  قم بتشغيل التطبيق:
    ```bash
    flutter run -d linux
    ```

## المميزات (Features)
يتمتع المشروع بمجموعة من الميزات القوية:

1.  **إدارة الأيقونات**:
    - دعم السحب والإفلات (Drag & Drop) للملفات من وإلى سطح المكتب.
    - ترتيب حر للأيقونات مع خاصية المحاذاة التلقائية للشبكة (Grid Snapping).
    - إنشاء مجلدات وملفات جديدة (Text, C, C++, Python, Dart, Markdown).

2.  **الخلفيات الذكية**:
    - دعم خلفيات الصور التقليدية (JPG, PNG).
    - **دعم حصري لخلفيات الفيديو** (MP4, MKV, WEBM) تعمل كخلفية حية لسطح المكتب.

3.  **قوائم السياق (Context Menus)**:
    - قائمة كليك يمين متكاملة لإنشاء الملفات، اللصق، تغيير الخلفية، وفتح الطرفية (Terminal).
    - خيارات لترتيب الأيقونات (حسب الاسم، النوع، الحجم، تاريخ التعديل).

4.  **التكامل مع النظام**:
    - فتح الملفات والمجلدات باستخدام التطبيقات الافتراضية للنظام.
    - "Open in Terminal" يدعم معظم الطرفيات الشهيرة (GNOME Terminal, Konsole, Kitty, Alacritty, etc.).
    - توليد مصغرات (Thumbnails) للصور والفيديوهات.

## الهيكلية البرمجية (Architecture)
يتبع المشروع نمط **Clean Architecture** مع **BLoC Pattern** لفصل المنطق عن الواجهة:

- **Presentation Layer**: تحتوي على واجهات المستخدم (Widgets) و BLoC (مثل `DesktopManagerBloc`) لإدارة الحالة.
- **Domain Layer**: تحتوي على التعريفات المجردة (Interfaces/Repositories) وقواعد العمل.
- **Data Layer**: تحتوي على تنفيذ المستودعات (Repositories Implementation) والتعامل المباشر مع نظام الملفات.

الملف الرئيسي `lib/main.dart` يحتوي على نقطة الدخول وواجهة سطح المكتب الرئيسية `DesktopView`.

## تحليل المشروع (Project Analysis)

### نقاط القوة (Strengths)
- **واجهة عصرية**: استخدام Flutter يوفر واجهة سلسة وجميلة مقارنة ببعض الواجهات التقليدية.
- **خلفيات الفيديو**: ميزة مدمجة قوية لا تتوفر عادة بشكل افتراضي.
- **كود نظيف**: هيكلية الكود منظمة وسهلة الصيانة.

### نقاط الضعف (Weaknesses)
- **الاعتماديات**: يتطلب تثبيت مكتبات خارجية (`libmpv`) مما قد يصعب التثبيت على المبتدئين.
- **معالجة الأخطاء**: يحتاج تحسينًا في التعامل مع الأخطاء (Error Handling) حيث يتم تجاهل بعض الاستثناءات حاليًا.
- **الميزات الناقصة**: يفتقر لبعض ميزات بيئات سطح المكتب الكاملة مثل شريط المهام (Taskbar) ودعم الشاشات المتعددة المتقدم.

### مجالات التحسين (Areas for Improvement)
1.  **تحسين الأداء**: ضبط استهلاك الذاكرة خاصة عند عرض الكثير من الصور المصغرة.
2.  **التعريب (Localization)**: دعم لغات متعددة (مثل العربية) في الواجهة بدلاً من النصوص الإنجليزية الثابتة.
3.  **الإعدادات**: إضافة واجهة إعدادات لتخصيص حجم الشبكة، وسلوك الخلفية، والخطوط.
